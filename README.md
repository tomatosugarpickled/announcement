# GlobalGates 담당 기능 발표 흐름

## 담당 범위

GlobalGates 프로젝트에서 사용자의 가입부터 로그인, 마이페이지 이용, 설정 관리까지 이어지는 계정 중심 기능을 담당했습니다.

이번 발표에서는 AI 기능은 제외하고 아래 세 가지 흐름을 중심으로 설명합니다.

- 로그인 / 회원가입
- 마이페이지
- 설정

---

## 1. 로그인 / 회원가입

사용자가 서비스를 시작하는 첫 진입 흐름입니다.

회원가입은 단순히 정보를 저장하는 기능이 아니라, 실제 서비스에서 필요한 검증 단계를 거쳐 안전하게 계정을 생성하도록 구성했습니다.

### 주요 기능

- 일반 회원가입
- 이메일, 전화번호, 핸들 중복 검사
- 회사명, 사업자번호 중복 검사
- SMS 인증
- 이메일 인증
- 프로필 이미지 업로드
- 일반 로그인
- JWT 기반 인증
- 비활성 계정 재활성화
- OAuth 신규 회원 추가 정보 입력

### 구현 흐름

1. 사용자가 회원가입 화면에서 기본 정보를 입력합니다.
2. 이메일, 전화번호, 핸들, 회사명, 사업자번호를 각각 중복 검사합니다.
3. 전화번호 또는 이메일 인증을 통해 실제 사용 가능한 연락처인지 확인합니다.
4. 프로필 이미지가 있으면 파일 업로드까지 함께 처리합니다.
5. 비밀번호는 암호화해서 저장합니다.
6. 로그인 성공 시 JWT access token과 refresh token을 발급합니다.
7. 이후 인증이 필요한 기능은 토큰을 기반으로 현재 로그인 사용자를 식별합니다.

### JWT를 사용한 이유

JWT를 사용한 가장 큰 이유는 보안입니다.

마이페이지 조회, 프로필 수정, 설정 변경, 상품 등록 같은 기능은 반드시 현재 로그인한 사용자 본인을 기준으로 실행되어야 합니다.

JWT를 사용하면 서버는 요청마다 전달된 토큰의 서명을 검증해 위조 여부를 확인할 수 있습니다. 토큰이 유효한 경우에만 사용자 정보를 꺼내 인증 객체를 만들고, 인증이 필요한 API에 접근할 수 있도록 처리했습니다.

특히 설정 변경이나 프로필 수정처럼 민감한 기능에서는 프론트에서 `memberId`를 직접 받지 않고, JWT로 식별한 현재 로그인 사용자 정보를 기준으로 처리했습니다. 이렇게 하면 다른 사용자의 id를 임의로 넣어 계정 정보를 수정하려는 요청을 막을 수 있습니다.

access token과 refresh token도 역할을 분리했습니다.

- access token: 인증이 필요한 요청에서 사용자 식별
- refresh token: access token 재발급

두 토큰의 역할을 나누어 인증 흐름을 관리하고, 토큰 노출 시 피해 범위를 줄일 수 있도록 구성했습니다.

### OAuth 흐름

OAuth 사용자는 기존 회원인지 신규 회원인지에 따라 흐름을 나눴습니다.

- 기존 OAuth 연동 회원: 바로 로그인 처리 후 토큰 발급
- 신규 OAuth 사용자: 바로 회원 생성하지 않고 추가 정보 입력 화면으로 이동

SNS에서 받은 기본 정보만으로는 서비스에 필요한 사업자 정보, 핸들, 관심 카테고리 등이 부족하기 때문에 추가 가입 단계를 따로 구성했습니다.

### 주요 API

- `POST /api/member/join`
- `POST /api/member/login`
- `POST /api/member/oauth/join`
- `GET /api/member/check-email`
- `GET /api/member/check-phone`
- `GET /api/member/check-handle`
- `GET /api/member/check-company-name`
- `GET /api/member/check-business-number`
- `POST /api/messages/send`
- `POST /api/mail/send-code`
- `POST /api/member/reactivation/prepare`
- `POST /api/member/reactivation/complete`

### 발표 포인트

- 단계별 회원가입 검증
- SMS / 이메일 인증 흐름
- JWT 기반 인증 구조
- access token / refresh token 분리
- OAuth 신규 회원 추가 정보 입력 흐름
- 비활성 계정 재활성화 처리

## 2. 마이페이지

마이페이지는 사용자의 프로필과 활동 내역을 모아서 보여주는 개인 공간입니다.

내 페이지에서는 프로필 수정과 상품 등록이 가능하고, 다른 사용자의 페이지에서는 팔로우, 차단, 신고, 채팅 같은 관계 기능을 사용할 수 있도록 구성했습니다.

### 주요 기능

- 내 프로필 조회
- 타인 프로필 조회
- 프로필 이미지 / 배너 이미지 수정
- 내 상품 목록 조회
- 내 게시글 목록 조회
- 내 댓글 목록 조회
- 좋아요한 게시글 조회
- 견적 요청 요약 조회
- 상품 등록
- 상품 삭제
- 팔로우 / 언팔로우
- 차단
- 신고
- 채팅 진입

### 구현 흐름

1. 마이페이지에 진입하면 페이지 주인과 현재 로그인 사용자를 비교합니다.
2. 본인 페이지라면 프로필 수정, 상품 등록 등 관리 기능을 보여줍니다.
3. 타인 페이지라면 팔로우, 차단, 신고, 채팅 버튼을 보여줍니다.
4. 상품, 게시글, 댓글, 좋아요, 견적 요청은 각각 독립된 API로 조회합니다.
5. 프론트에서는 탭별 페이지 상태를 따로 관리해 한 탭의 스크롤 상태가 다른 탭에 영향을 주지 않도록 했습니다.
6. 프로필 수정 시 닉네임, 소개글, 프로필 이미지, 배너 이미지를 함께 변경할 수 있습니다.
7. 이미지 파일은 S3에 업로드하고, 화면에는 presigned URL로 표시합니다.
8. 상품 등록 중 이미지 업로드 이후 실패가 발생하면 이미 업로드된 파일을 정리해 고아 파일이 남지 않도록 처리했습니다.

### 내 페이지와 타인 페이지 분리

마이페이지는 같은 화면을 사용하지만, 현재 로그인 사용자와 페이지 주인이 같은지에 따라 동작이 달라집니다.

- 본인 페이지: 프로필 수정, 상품 등록, 내 활동 관리
- 타인 페이지: 팔로우, 차단, 신고, 채팅

이 분기를 통해 한 화면에서 개인 관리 기능과 사용자 간 관계 기능을 함께 처리했습니다.

### 주요 API

- `GET /mypage/mypage`
- `GET /api/mypage/products`
- `GET /api/mypage/profile/products`
- `GET /api/mypage/posts`
- `GET /api/mypage/profile/posts`
- `GET /api/mypage/replies`
- `GET /api/mypage/profile/replies`
- `GET /api/mypage/likes`
- `GET /api/mypage/estimations/summary`
- `GET /api/mypage/estimations/requested`
- `POST /api/mypage/products`
- `POST /api/mypage/products/delete`
- `POST /api/member/profile/update`
- `POST /api/main/follows`
- `POST /api/main/follows/{followerId}/{followingId}/delete`
- `POST /api/v1/blocks`
- `POST /api/main/reports`

### 발표 포인트

- 본인 / 타인 프로필 기능 분기
- 탭별 활동 내역 분리 조회
- 프로필 / 배너 이미지 업로드
- 상품 등록과 이미지 실패 처리
- 팔로우, 차단, 신고 등 관계 기능 연결

## 3. 설정

설정 페이지는 사용자가 가입 이후에도 자신의 계정 정보를 직접 관리할 수 있도록 만든 기능입니다.

설정 기능은 범위가 넓기 때문에 발표에서는 전체 메뉴를 모두 시연하지 않고, 계정 정보 변경, 보안 설정, 사용자 환경 관리로 압축해서 설명합니다.

### 주요 기능

- 핸들 변경
- 전화번호 변경
- 이메일 변경
- 비밀번호 확인
- 비밀번호 변경
- 계정 비활성화
- 푸시 알림 on/off
- 세부 알림 설정
- 언어 / 국가 변경
- 차단 목록 조회
- 차단 해제
- 구독 상태 조회 / 해지 연동

### 구현 흐름

1. 설정 페이지에서 현재 로그인 사용자의 계정 정보를 보여줍니다.
2. 핸들 변경 시 회원가입에서 사용한 중복 검사 API를 재사용합니다.
3. 전화번호와 이메일 변경 시 기존 SMS / 이메일 인증 흐름을 재사용합니다.
4. 비밀번호 변경은 현재 비밀번호 검증 후 새 비밀번호를 암호화해 저장합니다.
5. 계정 비활성화는 데이터를 삭제하지 않고 회원 상태를 변경하는 방식으로 처리합니다.
6. 알림 설정은 전체 푸시 설정과 세부 알림 설정을 분리했습니다.
7. 차단 목록은 마이페이지에서 차단한 사용자를 설정 페이지에서 다시 확인하고 해제할 수 있도록 연결했습니다.

### 보안 관점

설정 API는 프론트에서 수정 대상 `memberId`를 직접 받지 않습니다.

서버는 JWT로 인증된 현재 로그인 사용자 정보를 기준으로 수정 대상을 결정합니다. 이렇게 해서 사용자가 요청 값을 조작해 다른 사용자의 핸들, 이메일, 전화번호, 비밀번호 등을 변경하려는 시도를 막을 수 있습니다.

비밀번호 변경과 계정 비활성화처럼 민감한 기능은 현재 비밀번호 검증 단계를 거치도록 구성했습니다.

### 시연에서 압축할 설정 메뉴

설정은 기능이 많기 때문에 아래 세 가지만 시연합니다.

1. 핸들 변경
2. 알림 설정
3. 차단 목록

전화번호 변경, 이메일 변경, 비밀번호 변경, 계정 비활성화는 실제 상태가 바뀌거나 외부 인증 API에 의존하므로 발표에서는 구조 설명과 Swagger 캡처로 대체합니다.

### 주요 API

- `POST /api/settings/check-password`
- `POST /api/settings/password`
- `POST /api/settings/handle`
- `POST /api/settings/phone`
- `POST /api/settings/email`
- `POST /api/settings/language`
- `POST /api/settings/country`
- `POST /api/settings/deactivate`
- `POST /api/settings/notifications/push-enabled`
- `POST /api/settings/notifications/push-preferences`
- `GET /api/settings/blocks/list/{page}`
- `DELETE /api/v1/blocks`

### 발표 포인트

- 회원가입 검증 로직 재사용
- 로그인 사용자 기준으로만 계정 정보 변경
- 비밀번호 변경 / 계정 비활성화 구현
- 알림 설정을 전체 / 세부 옵션으로 분리
- 마이페이지 차단 기능과 설정 차단 목록 연결

7. 핸들 변경 흐름 확인
8. 알림 설정 확인
9. 차단 목록 확인
10. Swagger에서 API 구조 짧게 확인

- 이메일 인증 전체 실시간 진행
- OAuth 신규 가입 전체 흐름

- 계정 비활성화 실제 실행
- 구독 해지 실제 실행
- 데이터가 비어 있는 마이페이지 화면

## 발표 마무리 요약

이번 담당 범위는 사용자의 계정 생명주기 전반을 다룹니다.

회원가입에서는 중복 검사와 인증을 통해 안전하게 계정을 만들고, 로그인에서는 JWT를 사용해 인증이 필요한 요청에서 현재 사용자를 식별했습니다.

마이페이지에서는 사용자 본인의 활동 관리와 타인 프로필에서의 관계 기능을 구현했습니다.

설정에서는 가입 이후에도 사용자가 자신의 계정 정보, 보안 상태, 알림, 차단 목록을 직접 관리할 수 있도록 구성했습니다.
